# Dokumentasi API, spesifikasi, dan definisi API

Tiga istilah ini sering disamakan, padahal fokusnya berbeda.

## 1. Dokumentasi API

Dokumentasi API adalah penjelasan yang ditulis untuk **manusia**, terutama developer.

Isinya biasanya meliputi:

- cara mulai menggunakan API,
- daftar endpoint,
- parameter request,
- contoh response,
- format error,
- contoh kode,
- panduan autentikasi.

## 2. Spesifikasi API

Spesifikasi API menjelaskan bagaimana API seharusnya bekerja secara lebih formal.

Biasanya spesifikasi menjelaskan:

- endpoint,
- parameter,
- request body,
- response,
- tipe data,
- relasi antarobjek,
- aturan perilaku API.

Contoh bahasa spesifikasi yang sering disebut:

- OpenAPI Specification,
- RAML,
- API Blueprint.

## 3. Definisi API

Definisi API lebih ditujukan untuk **mesin** atau tools otomatis.

Definisi ini bisa dipakai untuk:

- generate dokumentasi otomatis,
- generate SDK,
- membuat mock server,
- pengujian otomatis,
- validasi struktur API.
- validasi struktur API.
