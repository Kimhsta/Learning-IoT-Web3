# REST vs SOAP

REST dan SOAP sama-sama dipakai untuk komunikasi antar sistem, tetapi pendekatannya berbeda.

| Aspek | REST | SOAP |
|---|---|---|
| Bentuk | Gaya arsitektur | Protokol |
| Tingkat kerumitan | Lebih sederhana | Lebih kompleks |
| Format data | Umumnya JSON atau XML | Umumnya XML |
| Kinerja | Lebih ringan | Lebih berat |
| Fleksibilitas | Tinggi | Lebih ketat |
| Cocok untuk | Web modern, mobile, IoT | Sistem enterprise yang formal |

## Penjelasan sederhananya

- **REST** cocok untuk kebutuhan modern yang butuh cepat, ringan, dan mudah dikembangkan.
- **SOAP** cocok untuk sistem yang membutuhkan aturan sangat ketat, struktur formal, dan standar enterprise tertentu.

Karena itu, dalam banyak aplikasi web modern, REST lebih sering dipilih.
