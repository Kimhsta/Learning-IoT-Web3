# Penutup

API adalah jembatan komunikasi antar aplikasi. REST adalah gaya arsitektur yang membantu kita mendesain komunikasi tersebut agar lebih rapi, ringan, dan konsisten. RESTful API adalah API yang benar-benar dibangun mengikuti prinsip-prinsip REST.

Selain memahami API dan REST, developer juga perlu memahami dokumentasi API, spesifikasi API, dan definisi API. Ketiganya sangat penting agar API tidak hanya bisa dibuat, tetapi juga mudah dipakai, dipelihara, dan dikembangkan.

## Poin penting yang perlu diingat

- API memungkinkan aplikasi saling berkomunikasi.
- REST adalah gaya arsitektur, bukan protokol.
- RESTful berarti mengikuti prinsip REST.
- RESTful API biasanya memakai HTTP dan sering menggunakan JSON.
- Resource diakses lewat URI.
- Aksi dilakukan dengan method HTTP.
- REST menekankan stateless, uniform interface, cacheable, dan client-server separation.
- REST umumnya lebih ringan dan fleksibel dibanding SOAP.
- Dokumentasi API sangat penting agar API mudah digunakan developer lain.
