# Drop-in Network dan Edge Computing

## Walk-in Networking

Jaringan nirkabel dapat dibangun dalam berbagai topologi dan konfigurasi. Sebagai contoh, menghubungkan sensor atau perangkat jarak jauh melalui link nirkabel dengan intelligent gateway yang memiliki opsi transmisi data melalui seluler, Wi-Fi, atau Ethernet adalah pendekatan yang terbukti berhasil. Data dikumpulkan dari perangkat atau sensor jarak jauh dan dikirim ke smart gateway melalui link nirkabel. Gateway kemudian mengirim data ke cloud untuk dianalisis atau dimonitor.

“Drop-in network” adalah pembuatan jaringan yang spesifik untuk aplikasi Anda. Dalam lingkungan SCADA atau telemetri, gateway diperkenalkan yang memiliki kemampuan backhaul dan dapat terhubung ke node RF jarak jauh untuk mengirim data bolak-balik.

Pada beberapa aplikasi jaringan offline, router seluler yang terhubung ke perangkat mungkin sudah cukup. Contohnya adalah kiosk atau terminal point-of-sale di toko retail yang menggunakan konektivitas seluler untuk memproses transaksi.

Keuntungan lain dari jaringan akses langsung adalah Anda tidak perlu bergantung pada jaringan yang sudah ada atau meminta persetujuan untuk bergabung dengannya. Mendapatkan izin untuk masuk ke seluruh jaringan pelanggan sering kali memakan waktu lebih lama daripada mendesain dan men-deploy jaringan Anda sendiri. Membangun drop-in network memberi Anda fleksibilitas untuk menentukan topologi mana yang paling cocok antara mesh, point-to-point, dan point-to-multipoint serta memungkinkan Anda mengontrol semua aspek jaringan.

Jaringan mesh dengan ribuan node yang sering berkomunikasi dengan gateway sangat baik untuk aplikasi solar field dan street lighting. Penggunaan radio jarak jauh 900 MHz dalam aplikasi PTP atau PTMP dengan topologi jaringan direct-attach dapat membantu mencakup perangkat yang tersebar dalam jarak jauh, seperti di ladang minyak atau lingkungan pertanian.

## Edge intelligence

Kecerdasan di edge adalah salah satu tren IoT terbaru. Edge computing membantu membuat solusi lebih efisien dan mengurangi latensi saat mengirim data dari banyak perangkat ke cloud. Alih-alih mengirim semua data ke cloud untuk dianalisis dan ditindaklanjuti, keputusan dan pemrosesan data dapat terjadi di edge. Hal ini mengurangi lalu lintas pada jaringan dan memberikan peningkatan performa tambahan.

Daya tahan baterai juga merupakan faktor penting pada sensor yang di-deploy dari jarak jauh. Semakin banyak data yang ditransmisikan melalui jaringan, semakin besar konsumsi baterainya. Oleh karena itu, penting untuk mempertimbangkan seberapa sering perangkat Anda perlu melaporkan data dan menyesuaikan konektivitas modul Anda untuk performa yang optimal.

Kecerdasan di edge dapat membantu node sensor jarak jauh mengetahui kapan harus mengirim data berdasarkan perubahan kondisi, sehingga mengurangi konsumsi arus transmisi yang mahal dan laju data pada kasus konektivitas seluler.

Edge computing juga memungkinkan end device membuat keputusan bersyarat tanpa harus meminta perintah ke gateway, seperti anak kecil yang terus-menerus meminta izin kepada orang tuanya untuk melakukan sesuatu. Jika analogi itu dilanjutkan, bila Anda tidak ingin si anak meminta izin untuk segala hal, Anda dapat menambahkan sedikit kecerdasan pada perangkat agar dapat menangani proses-proses yang tidak kritis.
